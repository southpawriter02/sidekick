# Sidekick v0.4.2 Changelog ‚Äî Bookmark Workspaces

> **Release Date:** 2026-02-04  
> **Phase:** Navigation & Productivity (v0.4.x)

---

## Summary

This release adds **named bookmark/breakpoint workspaces** to Sidekick. Save and restore collections of bookmarks and breakpoints for different tasks or debugging sessions.

---

## New Features

### üìö Bookmark Workspaces

- **Named Collections:** Save sets of bookmarks and breakpoints as named workspaces
- **Quick Restore:** Switch between debugging contexts with one click
- **Persistent Storage:** Workspaces survive IDE restarts (stored in `sidekick-workspaces.xml`)
- **Import/Export:** Share workspace configurations across machines

---

## Components Added

### Models (`WorkspaceModels.kt`)

| Type | Description |
|------|-------------|
| `BookmarkWorkspace` | Named collection with bookmarks, breakpoints, timestamps |
| `SavedBookmark` | Bookmark data: file, line, mnemonic, description |
| `SavedBreakpoint` | Breakpoint data: file, line, condition, log expression |
| `WorkspaceExport` | Import/export format with version compatibility |
| `WorkspaceResult` | Sealed class for operation outcomes |

### Service (`WorkspaceService.kt`)

| Method | Description |
|--------|-------------|
| `createWorkspace()` | Creates a new named workspace |
| `saveCurrentState()` | Captures current bookmarks/breakpoints to workspace |
| `restoreWorkspace()` | Restores bookmarks/breakpoints from workspace |
| `deleteWorkspace()` | Removes a workspace |
| `exportWorkspaces()` | Exports all workspaces |
| `importWorkspaces()` | Imports from export format |

### Tool Window (`WorkspaceToolWindow.kt`)

| Component | Description |
|-----------|-------------|
| `WorkspacePanel` | UI with table view and action buttons |
| `WorkspaceTableModel` | Table model for workspace list |

### Actions

| Action | Description |
|--------|-------------|
| `CreateWorkspaceAction` | Create new workspace via menu |
| `SaveToWorkspaceAction` | Save current state to selected workspace |
| `RestoreWorkspaceAction` | Restore from selected workspace |

## Workspace Persistence

Workspaces are stored in your project's `.idea` directory:

```
.idea/sidekick-workspaces.xml
```

This file can be version-controlled to share workspaces with your team.
